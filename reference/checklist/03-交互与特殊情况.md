# 三、交互过程&特殊情况维度

> P1 = 必查项 | P2 = 建议项（重要需求或存疑时检查）

## 3.1 业务流程（P1）

- 流程是否涉及多角色？多端？线上线下场景？
- 该流程在大流程中处于哪个环节？上下游衔接流程是什么？
- 是否有展示初始状态？
- **B 端特别关注**：B 端多角色协作场景（如采购、仓储、财务等角色的权限差异）

## 3.2 新功能引导（P2）

- 新功能是否需要引导？引导是否清晰有效？
- 引导触发的场景和规则是否合适？如何消失？

## 3.3 核心流程（P1）

- 核心流程是否完整？流程链路是否过长过深？
- 操作动线是否合理？相似任务路径是否一致？
- 任务触发 / 返回 / 下一步 / 结束是否符合用户预期？
- **B 端特别关注**：批量操作的流程是否清晰？多步骤任务是否有进度指示？

## 3.4 页面跳转（P1）

- 跳转新页面？当前页面刷新？弹窗？判断依据：
  - 是否有主线 / 支线流程？
  - 内容量的多寡？
  - 是否打断工作流？
  - 是否需要跨页对照信息？

## 3.5 导航返回（P1）

- 是否能返回上一级？通过什么返回？
- 是否能精准返回某一级？

## 3.6 操作效率（P1）

- 操作步骤能否缩短？
- 重要 / 复杂操作是否提供说明？
- **B 端特别关注**：高频操作路径是否足够短？是否支持快捷操作 / 快捷键？

## 3.7 操作结果（P1）

- 操作完成后，用户是否清楚操作结果？
- 用户是否知道下一步该做什么？

## 3.8 页面定位（P1）

- 用户是否能识别自己在哪一级？了解页面层级关系？
- 跳转入口与页面标题是否一致？

## 3.9 界面反馈（P1）

**操作前（P1）**

- 操作前用户能预期会发生什么？
- 文案 / 指引是否明确，设计样式是否符合用户习惯？

**操作反馈（P1）**

- 强提示还是弱提示？用户是否清楚异常原因？
- 用户是否清楚接下来要做什么？

**容错性（P1）**

- 操作是否可逆？危险操作是否有二次确认？

**表单校验（P2）**

- 前端 / 后台的校验逻辑是什么？（实时校验还是提交后校验）
- Error 和 Warning 提示样式是否符合规范？

## 3.10 加载/上传/下载（P2）

- 网络错误 / 操作异常 / 加载超时如何提示？
- 成功 / 失败 / 部分成功 / 部分失败如何提示？
- **B 端特别关注**：批量操作的部分成功 / 失败状态的处理

## 3.11 数据加载（P2）

- 加载方式：同步还是异步？
- 复杂页面是否有分区域加载？是否有占位符加载（Skeleton）？
- 预览、保存、提交完成时间超过 3s 时是否有加载过渡态？
