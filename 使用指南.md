## 1. 什么是 Design-Selfcheck？

**Design-Selfcheck** 是一个运行在 Cursor AI 中的「设计稿自查技能」。你只需要把 Figma 链接发给它，它会：

- 自动读取你的设计稿内容（通过 Figma MCP）
- 向你收集需求背景（PRD 或对话形式）
- 按照专业的 B 端设计自查清单（需求、框架、交互、界面、交付五大维度）逐项分析
- 输出 **PASS / WARN / FAIL** 分级的结构化评审报告
- 给出「紧急必改」→「建议修改」→「可选优化」三档优先级建议

**适合场景：**

| 场景 | 说明 |
|------|------|
| 交付前自查 | 需求评审结束、设计稿完成后，提交给研发前做最后把关 |
| 设计走查准备 | 跟研发走查前提前发现明显问题 |
| 快速学习提升 | 通过报告里的建议，学习 B 端设计规范 |

## 3. 连接 Figma MCP

MCP（Model Context Protocol）是让 AI 直接读取你 Figma 文件的桥梁。没有它，AI 就看不到你的设计稿。

以下内容与 Cursor 官方文档中关于 **Figma MCP** 的推荐做法一致（可在 Cursor 帮助中心搜索 *Figma*、*MCP*）。

### 推荐方式：安装 Figma 插件（一键）

在 Cursor 的 **Agent 对话**里输入：

```
/add-plugin figma
```

按提示完成安装。该插件会一并配置好 Figma MCP，并附带常用工作流相关的 Agent Skills、Code Connect、设计系统规则与资源处理规则等，**优先用这种方式即可**。

### 手动配置（可选）

若你希望自己在设置里添加全局 MCP，可按下面步骤操作：

1. 打开 **Cursor** → **Settings** → **Cursor Settings**
2. 进入 **MCP** 标签页
3. 点击 **「+ Add new global MCP server」**
4. 填入并保存以下配置（`url` 须与官方一致）：

```json
{
  "mcpServers": {
    "figma": {
      "url": "https://mcp.figma.com/mcp"
    }
  }
}
```

更细的参数说明与排障，请以 Cursor 官方文档为准。

### 如何确认连接成功

**信号一**：在 Agent 界面侧栏或 MCP 相关区域能看到 **figma** 服务已启用（具体 UI 以当前 Cursor 版本为准）。

**信号二**：在 **Agent** 对话框里发一条与 Figma 相关的指令（例如附上你的 `figma.com/design/...` 链接，或让 AI 说明当前能否访问 Figma 工具），若 AI 能正常基于设计稿响应，即表示 MCP 工作正常。

> 若连接异常：先确认上述 JSON 中的 `url` 为 `https://mcp.figma.com/mcp`，再检查网络与 Figma 账号登录状态；仍无法解决时查阅 Cursor 官方文档中的 MCP 章节。

## 4. 安装 SKILL.md 到 Cursor 目录

Cursor 的「技能」本质上是一个放在特定位置的 `SKILL.md` 文件。Cursor 在对话时会自动加载已安装的技能。

### 4.1 创建技能目录

打开终端，运行以下命令：

```bash
# 创建 design-selfcheck 技能目录
mkdir -p ~/.cursor/skills/design-selfcheck
```

### 4.2 下载 SKILL.md 文件

**方式 A：从分享链接下载**

如果有人分享了 `SKILL.md` 文件给你，下载到本地，然后：

```bash
# 把文件复制到技能目录（替换路径为你实际的下载路径）
cp ~/Downloads/SKILL.md ~/.cursor/skills/design-selfcheck/SKILL.md
```

**方式 B：直接在 Cursor 中创建**

1. 打开 Cursor，按 `Cmd + O` 打开文件
2. 导航到 `~/.cursor/skills/design-selfcheck/`
3. 新建文件 `SKILL.md`
4. 将 SKILL.md 的完整内容粘贴进去，保存

**清单文件说明：** 当前版本的自查条目拆在 `reference/checklist/` 目录下。若你从仓库或他人处复制本技能，请**整个 `design-selfcheck` 文件夹**一并复制（勿只拷贝单个 `SKILL.md`），否则 Agent 在 Phase 3 无法读取完整清单。完整单文件旧版备份为同目录下的 `SKILL.full-backup.md`（可选）。

### 4.3 验证安装是否成功

打开 Cursor 的技能管理界面：

1. 在 Cursor 设置 → **Features** → **Skills** 中查看是否出现 `design-selfcheck`
2. 或者直接在 Agent 对话中发送：
   ```
   你有哪些已安装的技能？
   ```
   AI 会列出所有已加载的技能，确认 `design-selfcheck` 在列表中

## 5. 如何调用技能（三种方式）

### 方式 A：发送 Figma 链接（最直接）

直接在 Cursor Agent 对话框中粘贴你的 Figma 链接：

```
https://www.figma.com/design/xxxxxxxxxx/项目名称?node-id=xxx
```

AI 会自动识别这是一个设计稿链接，触发 design-selfcheck 流程。

### 方式 B：使用触发关键词

在消息中包含以下任一关键词即可触发：

- `帮我自查这个设计稿`
- `review 一下我的设计`
- `设计自查`
- `帮我查设计`

**示例：**
```
帮我自查一下这个设计稿 https://www.figma.com/design/xxx/...
这是供应链采购模块的新需求，主要面向采购专员
```

### 方式 C：先发 PRD 再发设计链接

对于有 PRD 文档的需求，先发 PRD 效果更好：

```
【第一条消息】
这是这次需求的 PRD 文档（粘贴或上传 PDF）

【第二条消息】
设计稿在这里，帮我自查：
https://www.figma.com/design/xxx/...
```

## 6. 一次完整的自查示例

以下展示从发送链接到拿到报告的完整流程：

### Step 1：发送 Figma 链接

你发送：
```
帮我 review 一下这个设计稿
https://www.figma.com/design/abc123/采购单管理?node-id=1-1
```

### Step 2：AI 收集背景信息

AI 回复：
```
我需要了解几个背景信息，才能更准确地帮你自查：

1. 这个需求是为了解决什么问题？业务目标是什么？
2. 主要面向哪类用户？（角色/职能/使用场景）
3. 这次设计最希望达成什么效果？
4. 涉及哪些主要功能模块？
5. 有没有已知的技术限制或品牌规范要求？
```

### Step 3：你回答背景问题

你回复：
```
1. 采购单审批流程太繁琐，想优化到一个页面完成所有操作
2. 主要用户是采购专员，每天要处理 50+ 条采购单
3. 希望提升审批效率，减少页面跳转次数
4. 涉及：采购单列表、批量审批、异常处理
5. 需要兼容 1280px 屏幕，遵循公司内部 Design System
```

### Step 4：收到评审报告

AI 生成结构化报告，包含：

```markdown
# 设计方案评审报告

## 一、需求背景摘要
产品目标：优化采购单审批流程...
核心用户：采购专员，高频、批量操作场景...

## 二、设计推导
用户目标：快速完成多条采购单的批量审批...

## 三、设计缺陷分析

### 用户卡点
**[严重] 批量操作入口不明显**：
当前设计中批量审批功能藏在二级菜单，
采购专员在高频操作场景下需要多次点击才能触达，
建议将批量操作提升为列表页常驻操作区...

**[一般] 缺少「部分失败」状态设计**：
批量提交时若部分审批失败，
当前设计无法区分成功/失败条目...

## 四、其他设计方向参考
...

## 五、综合建议

### 紧急重要（本次必改）
1. 将批量操作按钮提升至表格顶部操作栏...

### 重要不紧急（下版本跟进）
1. 增加审批进度可视化...
```

## 7. 常见问题 FAQ

**Q：AI 说"无法访问 Figma 链接"怎么办？**

A：检查以下几点：
1. 是否已按官方方式完成 Figma MCP（推荐 `/add-plugin figma`，或 MCP 中 `url` 为 `https://mcp.figma.com/mcp`）
2. Figma 账号是否已登录，当前文件是否有访问权限（团队库 / 分享链接等）
3. 网络是否正常；必要时在 **Cursor Settings → MCP** 中确认 **figma** 服务状态并重试

**Q：没有 Figma 链接，只有截图可以用吗？**

A：可以！直接把设计稿截图拖入对话框，AI 会基于截图进行分析，但会在报告中注明"基于截图推断"，准确度略低于直接读取 Figma 文件。

**Q：自查清单是什么？我需要自己对照吗？**

A：不需要。SKILL.md 中内置了完整的自查清单（涵盖需求分析、框架布局、交互、界面、交付五大维度），AI 会在内部自动对照，报告中不会出现条目编号，全部用自然语言输出，直接可读可用。

**Q：可以自定义自查重点吗？**

A：可以。在发送 Figma 链接时附上特别说明：
```
帮我重点关注批量操作和权限相关的设计问题
https://www.figma.com/design/xxx/...
```

**Q：SKILL.md 文件在哪里找？**

A：SKILL.md 文件位于 `~/.cursor/skills/design-selfcheck/SKILL.md`，如果你安装了技能，可以用 Cursor 直接打开查看；也可以由技能作者通过文件分享给你。

**Q：免费版 Cursor 能用吗？**

A：公司内一般通过 [IT Center](https://itcenter.sea.com/) 开通 **企业权限**，按席位与策略使用即可。若使用个人免费版，会有月度额度等限制；复杂设计稿分析可能消耗较多 token，具体以 Cursor 与 IT 政策为准。
